# Supabase SQL Setup and RLS Policies

Execute the following SQL in your Supabase SQL Editor to create the necessary tables and enable Row Level Security (RLS) policies required for PasalHub sync.

## 1. Table Definitions

If you haven't created the tables yet, use this script. Note that `profiles` is linked to `auth.users`.

```sql
-- Profiles table (Extends Auth)
CREATE TABLE IF NOT EXISTS public.profiles (
  id UUID REFERENCES auth.users ON DELETE CASCADE PRIMARY KEY,
  email TEXT,
  name TEXT,
  address TEXT,
  date_of_birth TEXT,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
);

-- Products table
CREATE TABLE IF NOT EXISTS public.products (
  id BIGINT PRIMARY KEY,
  title TEXT NOT NULL,
  price DECIMAL NOT NULL,
  description TEXT,
  category TEXT,
  image TEXT,
  embedding VECTOR(384) -- Optional: for visual search if using pgvector
);

-- Cart table
CREATE TABLE IF NOT EXISTS public.cart (
  user_id UUID REFERENCES auth.users ON DELETE CASCADE,
  product_id BIGINT REFERENCES public.products(id) ON DELETE CASCADE,
  quantity INTEGER DEFAULT 1,
  PRIMARY KEY (user_id, product_id)
);

-- Favorites table
CREATE TABLE IF NOT EXISTS public.favorites (
  user_id UUID REFERENCES auth.users ON DELETE CASCADE,
  product_id BIGINT REFERENCES public.products(id) ON DELETE CASCADE,
  PRIMARY KEY (user_id, product_id)
);

-- Orders table
CREATE TABLE IF NOT EXISTS public.orders (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users ON DELETE CASCADE,
  total_amount DECIMAL NOT NULL,
  items_summary TEXT,
  status TEXT DEFAULT 'Placing',
  address TEXT,
  date BIGINT, -- Storing timestamp as bigint to match local Long
  quantity INTEGER,
  seller TEXT
);
```

## 2. Enable Row Level Security (RLS)

```sql
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.products ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.cart ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.favorites ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.orders ENABLE ROW LEVEL SECURITY;
```

## 3. RLS Policies

These policies ensure that users can only access their own data, while products are readable by everyone.

### Profiles
```sql
CREATE POLICY "Users can view their own profile" ON public.profiles
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users can update their own profile" ON public.profiles
  FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Users can insert their own profile" ON public.profiles
  FOR INSERT WITH CHECK (auth.uid() = id);
```

### Products
```sql
CREATE POLICY "Products are viewable by everyone" ON public.products
  FOR SELECT USING (true);
```

### Cart
```sql
CREATE POLICY "Users can manage their own cart" ON public.cart
  FOR ALL USING (auth.uid() = user_id);
```

### Favorites
```sql
CREATE POLICY "Users can manage their own favorites" ON public.favorites
  FOR ALL USING (auth.uid() = user_id);
```

### Orders
```sql
CREATE POLICY "Users can manage their own orders" ON public.orders
  FOR ALL USING (auth.uid() = user_id);
```

> [!TIP]
> After executing these, make sure you have some data in the `products` table so the app can fetch them! You can import a CSV or manually insert some items matching the `ProductEntity` structure.
